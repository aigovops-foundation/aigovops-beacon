What to do

    What this checks

    The lab needs each of these. Here's why.

    The Framework Lab is the "evidence model in your browser" — keys generated locally, receipts signed locally, bundles downloaded locally. Every check above corresponds to one feature you'll use during the 30-minute walk-through.

    • Modern browser + JavaScript — table-stakes; if these fail the lab won't load at all.
    • Web Crypto API — used to fingerprint your Ed25519 public key for the certificate diploma.
    • localStorage — saves your progress between sections and stores quiz answers. If disabled, the lab still works but progress resets on every reload.
    • tweetnacl + JSZip — the Ed25519 signing library and the bundle-zipper. Both load from cdn.jsdelivr.net. If your network blocks the CDN, the sandbox in Section 4 of Level 100 won't work.
    • CORS to bobrapp.github.io — the 100-failure deep-dive table in Level 200 fetches the dataset from the canonical Beacon repo. If this fails, Level 200 still works for everything except the deep-dive.
    • Live sign + verify — end-to-end smoke test. Generates an Ed25519 keypair, signs canonical bytes, verifies the signature. If this passes, the Level 100 sandbox will work.
